Beginner Level

🌱 Beginner SQL

Start from absolute zero. No prior SQL knowledge needed. Every concept explained simply.

1. What is a Database?

A database is an organized collection of structured data stored electronically. Think of it like a digital filing cabinet where you store information in a very organized way so you can find it quickly.

📘 Real World AnalogyThink of a school's student records room. Each drawer = a table, each file = a row, each field on the form = a column.

RDBMSRelational DBMSStores data in related tablesMySQL, PostgreSQL, Oracle

2. What is SQL?

SQL (Structured Query Language) is the standard language used to communicate with relational databases. You use SQL to create, read, update, and delete data.

Types of SQL Commands

3. Tables, Rows, and Columns

Data in a relational database is stored in tables. Each table has:

  • Columns — the fields/attributes (like Name, Age, Salary)
  • Rows — each record of data (like one employee's details)

Primary Key & Foreign Key

🔑 Primary Key

A column (or set of columns) that uniquely identifies each row in a table.

  • Must be UNIQUE
  • Cannot be NULL
  • Only ONE per table
  • Example: emp_id

🔗 Foreign Key

A column that references the Primary Key of another table — creates a relationship.

  • Can have duplicates
  • Can be NULL
  • Links two tables
  • Example: dept_id in Employees

Creating and Using a Database

SQL 
CREATE TABLE employees (

SELECT — Retrieve Data

SQL 
-- Select ALL columns
SELECT * FROM employees;

-- Select specific columns
SELECT emp_name, department, salary
FROM employees;

-- SELECT DISTINCT — remove duplicates
SELECT DISTINCT department
FROM employees;
✅ Best PracticeAvoid SELECT * in production code. Always specify the columns you need — it's faster and clearer.

6. WHERE Clause — Filtering Data

The WHERE clause filters rows based on conditions.

SQL 
-- Basic WHERE
SELECT * FROM employees
WHERE department = 'Engineering';

-- AND — both conditions must be true
SELECT * FROM employees
WHERE department = 'Engineering' AND salary > 80000;

-- OR — either condition can be true
SELECT * FROM employees
WHERE department = 'HR' OR department = 'Marketing';

-- NOT
SELECT * FROM employees
WHERE NOT department = 'HR';

-- BETWEEN — range (inclusive)
SELECT * FROM employees
WHERE salary BETWEEN 70000 AND 90000;

-- IN — match any value in a list
SELECT * FROM employees
WHERE department IN ('Engineering', 'Marketing', 'Sales');

-- LIKE — pattern matching
SELECT * FROM employees
WHERE emp_name LIKE 'A%';     -- starts with A

SELECT * FROM employees
WHERE emp_name LIKE '%son';   -- ends with son

SELECT * FROM employees
WHERE emp_name LIKE '%ar%';   -- contains 'ar'

-- IS NULL — check for missing values
SELECT * FROM employees
WHERE manager_id IS NULL;

-- IS NOT NULL
SELECT * FROM employees
WHERE manager_id IS NOT NULL;
📘 LIKE Wildcards % matches any number of characters (including zero)
_ matches exactly ONE character
Example: LIKE '_ob%' matches "Bob", "Rob", "Job..."

7. ORDER BY and LIMIT

SQL 
-- Sort by salary highest to lowest
SELECT emp_name, salary
FROM employees
ORDER BY salary DESC;

-- Sort alphabetically
SELECT emp_name
FROM employees
ORDER BY emp_name ASC;

-- LIMIT — get only top N rows
SELECT emp_name, salary
FROM employees
ORDER BY salary DESC
LIMIT 5;  -- Top 5 highest paid

-- LIMIT with OFFSET — pagination
SELECT * FROM employees
LIMIT 10 OFFSET 20;  -- Page 3 (rows 21-30)

Intermediate Level

⚡ Intermediate SQL

1. GROUP BY & Aggregate Functions

Aggregate functions perform calculations on groups of rows and return a single value.

FunctionPurposeExample
COUNT()Count rowsCOUNT(*) → 10
SUM()Total of valuesSUM(salary) → 850000
AVG()Average valueAVG(salary) → 78000
MIN()Smallest valueMIN(salary) → 55000
MAX()Largest valueMAX(salary) → 95000
SQL 
-- Count total employees
SELECT COUNT(*) AS total_employees FROM employees;

-- Average salary per department
SELECT department,
       COUNT(*) AS num_employees,
       AVG(salary) AS avg_salary,
       MAX(salary) AS max_salary,
       MIN(salary) AS min_salary
FROM employees
GROUP BY department;

-- HAVING — filter groups (like WHERE but for groups)
SELECT department, COUNT(*) AS num_employees
FROM employees
GROUP BY department
HAVING COUNT(*) > 2;  -- Only depts with >2 employees
⚠️ WHERE vs HAVING WHERE filters individual rows before grouping.
HAVING filters groups after GROUP BY.
You cannot use aggregate functions (SUM, COUNT...) in WHERE.

2. JOINs — Combining Tables

JOINs combine rows from two or more tables based on a related column.

Table A
Table B
INNER JOIN
Only matching rows
LEFT JOIN
All A + matching B
RIGHT JOIN
All B + matching A
FULL JOIN
All rows from both
SELF JOIN
Table joins itself
CROSS JOIN
Every combo (A×B)
SQL — All JOIN Types 
-- INNER JOIN — only rows that match in both tables
SELECT e.emp_name, e.salary, d.dept_name
FROM employees e
INNER JOIN departments d
    ON e.dept_id = d.dept_id;

-- LEFT JOIN — all employees, even without a department
SELECT e.emp_name, d.dept_name
FROM employees e
LEFT JOIN departments d
    ON e.dept_id = d.dept_id;

-- RIGHT JOIN — all departments, even with no employees
SELECT e.emp_name, d.dept_name
FROM employees e
RIGHT JOIN departments d
    ON e.dept_id = d.dept_id;

-- FULL OUTER JOIN — everything from both tables
SELECT e.emp_name, d.dept_name
FROM employees e
FULL OUTER JOIN departments d
    ON e.dept_id = d.dept_id;

-- SELF JOIN — employees with their manager names
SELECT e.emp_name AS employee,
       m.emp_name AS manager
FROM employees e
LEFT JOIN employees m
    ON e.manager_id = m.emp_id;

-- CROSS JOIN — every combination of rows
SELECT e.emp_name, d.dept_name
FROM employees e
CROSS JOIN departments d;

3. Subqueries

A subquery is a query nested inside another query.

SQL 
-- Single-row subquery — employees earning more than average
SELECT emp_name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees);

-- Multi-row subquery — employees in Engineering or Sales
SELECT emp_name
FROM employees
WHERE dept_id IN (
    SELECT dept_id
    FROM departments
    WHERE dept_name IN ('Engineering', 'Sales')
);

-- Correlated subquery — employees who earn more than their dept average
SELECT emp_name, salary, department
FROM employees e1
WHERE salary > (
    SELECT AVG(salary)
    FROM employees e2
    WHERE e2.department = e1.department
);

-- EXISTS — departments that have employees
SELECT dept_name
FROM departments d
WHERE EXISTS (
    SELECT 1
    FROM employees e
    WHERE e.dept_id = d.dept_id
);

4. UNION & UNION ALL

SQL 
-- UNION — combine results, removes duplicates
SELECT emp_name FROM employees WHERE department = 'Engineering'
UNION
SELECT emp_name FROM employees WHERE salary > 80000;

-- UNION ALL — keeps duplicates, faster
SELECT emp_name FROM employees WHERE department = 'Engineering'
UNION ALL
SELECT emp_name FROM employees WHERE salary > 80000;

5. CASE WHEN — Conditional Logic

SQL 
-- Categorize employees by salary band
SELECT emp_name, salary,
    CASE
        WHEN salary >= 90000 THEN 'Senior'
        WHEN salary >= 70000 THEN 'Mid-Level'
        ELSE 'Junior'
    END AS level
FROM employees;
Advanced Level

🚀 Advanced SQL

1. Window Functions

Window functions perform calculations across a set of rows related to the current row, without collapsing rows like GROUP BY does.

📘 Key ConceptWindow functions use the OVER() clause. They return a value for each row while using data from other rows.
SQL — Window Function Syntax 
function_name() OVER (
    [PARTITION BY column]  -- like GROUP BY for windowing
    [ORDER BY column]      -- order within each partition
    [ROWS BETWEEN ...]     -- frame definition
)
SQL — Ranking Functions 
SELECT
    emp_name,
    department,
    salary,
    ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) AS row_num,
    RANK()       OVER (PARTITION BY department ORDER BY salary DESC) AS rank_num,
    DENSE_RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS dense_rank,
    NTILE(4)     OVER (ORDER BY salary DESC) AS quartile
FROM employees;

-- RANK vs DENSE_RANK:
-- Salaries: 90000, 85000, 85000, 72000
-- RANK:       1,     2,     2,     4   (gap after tie)
-- DENSE_RANK: 1,     2,     2,     3   (no gap)
SQL — LAG & LEAD (Compare Rows) 
SELECT
    emp_name,
    salary,
    LAG(salary, 1) OVER (ORDER BY salary) AS prev_salary,
    LEAD(salary, 1) OVER (ORDER BY salary) AS next_salary,
    salary - LAG(salary, 1) OVER (ORDER BY salary) AS salary_diff
FROM employees;

-- Running total (cumulative sum)
SELECT
    emp_name,
    salary,
    SUM(salary) OVER (ORDER BY emp_id) AS running_total
FROM employees;

Top N per Group — Classic Interview Query

SQL — Top 3 Salaries Per Department 
WITH ranked AS (
    SELECT
        emp_name,
        department,
        salary,
        DENSE_RANK() OVER (PARTITION BY department ORDER BY salary DESC) AS rnk
    FROM employees
)
SELECT emp_name, department, salary
FROM ranked
WHERE rnk <= 3;

2. Common Table Expressions (CTEs)

A CTE is a named temporary result set you can reference within a SELECT, INSERT, UPDATE, or DELETE. Makes complex queries readable.

SQL — Basic CTE 
-- CTE to find high-earning Engineering employees
WITH engineering_staff AS (
    SELECT emp_name, salary
    FROM employees
    WHERE department = 'Engineering'
),
high_earners AS (
    SELECT emp_name, salary
    FROM engineering_staff
    WHERE salary > 80000
)
SELECT * FROM high_earners;

-- Recursive CTE — org chart (employees + their managers)
WITH RECURSIVE org_chart AS (
    -- Anchor: top-level (no manager)
    SELECT emp_id, emp_name, manager_id, 0 AS level
    FROM employees
    WHERE manager_id IS NULL

    UNION ALL

    -- Recursive: employees reporting to someone
    SELECT e.emp_id, e.emp_name, e.manager_id, oc.level + 1
    FROM employees e
    INNER JOIN org_chart oc ON e.manager_id = oc.emp_id
)
SELECT * FROM org_chart ORDER BY level;

5. Transactions & ACID

A — Atomicity

All operations in a transaction succeed, or none do. No partial commits.

C — Consistency

Database stays in a valid state before and after the transaction.

I — Isolation

Transactions execute independently — one can't see another's partial changes.

D — Durability

Once committed, data persists permanently even after system failure.

SQL — Transactions 
START TRANSACTION;

    -- Transfer salary between accounts
    UPDATE employees SET salary = salary - 5000
    WHERE emp_id = 101;

    SAVEPOINT after_deduction;

    UPDATE employees SET salary = salary + 5000
    WHERE emp_id = 102;

COMMIT;  -- Save all changes permanently

-- If something went wrong:
ROLLBACK;  -- Undo everything back to START TRANSACTION
ROLLBACK TO SAVEPOINT after_deduction;  -- Go back to savepoint

-- Create stored procedure
DELIMITER //

Database Design

🗄️ Database Design

1. Normalization

Normalization is the process of organizing a database to reduce redundancy and improve data integrity.

1NF — First Normal Form

✅ Each column contains atomic (indivisible) values. ✅ No repeating groups. ✅ Each row is unique.

❌ NOT in 1NF
emp_id | emp_name | skills
101    | Alice    | Python, SQL, Java    ← multiple values in one cell
✅ In 1NF
emp_id | emp_name | skill
101    | Alice    | Python
101    | Alice    | SQL
101    | Alice    | Java

2NF — Second Normal Form

✅ Must be in 1NF. ✅ Every non-key column must depend on the entire primary key (no partial dependencies).

📘 Applies to composite primary keysIf PK = (order_id, product_id), then customer_name should NOT be in this table since it only depends on order_id, not the full PK.

3NF — Third Normal Form

✅ Must be in 2NF. ✅ No transitive dependencies (non-key columns should not depend on other non-key columns).

❌ Transitive Dependency
emp_id → dept_id → dept_name
dept_name depends on dept_id, not directly on emp_id
✅ Fixed — Split into two tables
employees:  emp_id, emp_name, dept_id
departments: dept_id, dept_name

BCNF — Boyce-Codd Normal Form

✅ Stronger version of 3NF. For every functional dependency X → Y, X must be a super key. Handles edge cases that 3NF misses.

3. Relationship Types

TypeDescriptionExampleImplementation
One-to-One (1:1)One row in A relates to exactly one in BEmployee ↔ PassportForeign key in either table

SQL — Many-to-Many with Junction Table 
-- Students can enroll in many courses, each course has many students
CREATE TABLE students (
    student_id INT PRIMARY KEY,
    name VARCHAR(100)
);

CREATE TABLE courses (
    course_id INT PRIMARY KEY,
    course_name VARCHAR(100)
);

-- Junction table to handle M:N
CREATE TABLE enrollments (
    student_id INT,
    course_id  INT,
    enroll_date DATE,
    PRIMARY KEY (student_id, course_id),
    FOREIGN KEY (student_id) REFERENCES students(student_id),
    FOREIGN KEY (course_id)  REFERENCES courses(course_id)
);
Interview Prep

💼 SQL Interview Questions

Top SQL interview questions with complete solutions. Practice these and you'll ace any SQL interview.

🟢 Easy Questions

1. Find the second highest salary Easy
Solution
-- Method 1: Using subquery
SELECT MAX(salary) AS second_highest
FROM employees
WHERE salary < (SELECT MAX(salary) FROM employees);

-- Method 2: Using DENSE_RANK (handles ties correctly)
SELECT salary AS second_highest
FROM (
    SELECT salary,
           DENSE_RANK() OVER (ORDER BY salary DESC) AS rnk
    FROM employees
) ranked
WHERE rnk = 2;
2. Find duplicate rows in a table Easy
Solution
-- Find duplicate emp_names
SELECT emp_name, COUNT(*) AS duplicate_count
FROM employees
GROUP BY emp_name
HAVING COUNT(*) > 1;
3. Delete duplicate rows, keep one Easy
Solution
-- Keep the row with the lowest emp_id
DELETE FROM employees
WHERE emp_id NOT IN (
    SELECT MIN(emp_id)
    FROM employees
    GROUP BY emp_name, department, salary
);
4. Nth highest salary (generalized) Easy
Solution
-- Get the Nth highest salary (e.g., N=3)
SELECT salary
FROM (
    SELECT salary,
           DENSE_RANK() OVER (ORDER BY salary DESC) AS rnk
    FROM employees
) ranked
WHERE rnk = 3;  -- Change 3 to N

🟡 Medium Questions

5. Employees earning more than their manager Medium
Solution
SELECT e.emp_name AS employee,
       e.salary AS emp_salary,
       m.emp_name AS manager,
       m.salary AS mgr_salary
FROM employees e
JOIN employees m ON e.manager_id = m.emp_id
WHERE e.salary > m.salary;
6. Top 3 employees per department Medium
Solution
WITH dept_ranked AS (
    SELECT emp_name, department, salary,
           DENSE_RANK() OVER (
               PARTITION BY department
               ORDER BY salary DESC
           ) AS rnk
    FROM employees
)
SELECT emp_name, department, salary
FROM dept_ranked
WHERE rnk <= 3
ORDER BY department, rnk;
7. Running total (cumulative sum) Medium
Solution
SELECT
    emp_name,
    salary,
    SUM(salary) OVER (
        ORDER BY emp_id
        ROWS BETWEEN UNBOUNDED PRECEDING AND CURRENT ROW
    ) AS running_total
FROM employees;

🔴 Hard Questions

8. Gaps and Islands Problem Hard

Find consecutive date ranges (islands) and gaps in date sequences.

Solution
-- Find consecutive login streaks
WITH dated AS (
    SELECT
        user_id,
        login_date,
        login_date - ROW_NUMBER() OVER (
            PARTITION BY user_id
            ORDER BY login_date
        ) * INTERVAL '1 day' AS grp
    FROM user_logins
)
SELECT
    user_id,
    MIN(login_date) AS streak_start,
    MAX(login_date) AS streak_end,
    COUNT(*) AS streak_length
FROM dated
GROUP BY user_id, grp
ORDER BY user_id, streak_start;
9. Pivot table — departments as columns Hard
Solution (MySQL)
SELECT
    SUM(CASE WHEN department = 'Engineering' THEN salary END) AS Engineering,
    SUM(CASE WHEN department = 'Marketing'   THEN salary END) AS Marketing,
    SUM(CASE WHEN department = 'HR'          THEN salary END) AS HR
FROM employees;
Practice

🧩 Practice Problems

Leetcode-style SQL problems. Try solving before revealing the solution!

🟢 Beginner Problems

Easy P1: List all employees with salary above average

Problem Statement

Write a SQL query to find all employees whose salary is above the company average salary. Return emp_name and salary.

Table: employees

emp_idemp_namedepartmentsalary
101AliceEngineering85000
102BobMarketing62000
103CarolEngineering91000
104DaveHR55000

Expected Output

emp_namesalary
Alice85000
Carol91000

Solution

SELECT emp_name, salary
FROM employees
WHERE salary > (SELECT AVG(salary) FROM employees)
ORDER BY salary DESC;
💡 ExplanationThe subquery calculates the average salary first, then the outer query compares each employee's salary to that average.
Easy P2: Count employees per department

Write a query to show the number of employees in each department, only for departments with more than 1 employee.

Solution
SELECT department, COUNT(*) AS employee_count
FROM employees
GROUP BY department
HAVING COUNT(*) > 1
ORDER BY employee_count DESC;

🟡 Intermediate Problems

Medium P3: Find customers who never placed an order

Find all customers who have not placed any orders.

Solution
-- Method 1: LEFT JOIN + IS NULL
SELECT c.customer_id, c.customer_name
FROM customers c
LEFT JOIN orders o ON c.customer_id = o.customer_id
WHERE o.order_id IS NULL;

-- Method 2: NOT EXISTS
SELECT customer_id, customer_name
FROM customers c
WHERE NOT EXISTS (
    SELECT 1 FROM orders
    WHERE customer_id = c.customer_id
);
Medium P4: Monthly sales report

Write a query to show total revenue per month from the orders table.

Solution
SELECT
    YEAR(order_date) AS year,
    MONTH(order_date) AS month,
    COUNT(*) AS total_orders,
    SUM(total_amount) AS total_revenue,
    AVG(total_amount) AS avg_order_value
FROM orders
GROUP BY YEAR(order_date), MONTH(order_date)
ORDER BY year, month;

🔴 Advanced Problems

Hard P5: Median salary per department

Calculate the median salary for each department.

Solution
WITH ranked AS (
    SELECT
        department,
        salary,
        ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary) AS rn,
        COUNT(*) OVER (PARTITION BY department) AS cnt
    FROM employees
)
SELECT
    department,
    AVG(salary) AS median_salary
FROM ranked
WHERE rn IN (FLOOR((cnt + 1) / 2), CEIL((cnt + 1) / 2))
GROUP BY department;

CREATE TABLE watch_history (
    watch_id    INT PRIMARY KEY AUTO_INCREMENT,
    user_id     INT,
    content_id  INT,
    watched_at  DATETIME,
    watch_pct   DECIMAL(5,2),  -- % of content watched
    FOREIGN KEY (user_id) REFERENCES users(user_id),
    FOREIGN KEY (content_id) REFERENCES content(content_id)
);

-- Top 10 most watched shows this month
SELECT
    c.title,
    c.genre,
    COUNT(wh.watch_id) AS view_count,
    AVG(wh.watch_pct) AS avg_completion
FROM content c
JOIN watch_history wh ON c.content_id = wh.content_id
WHERE MONTH(wh.watched_at) = MONTH(NOW())
GROUP BY c.content_id, c.title, c.genre
ORDER BY view_count DESC
LIMIT 10;

📊 Case Study 1: E-commerce Sales Analysis

Business Problem: The CEO wants to know which product categories are driving the most revenue and which customer segments are most valuable.

Solution Query
-- Revenue by category with month-over-month growth
WITH monthly_revenue AS (
    SELECT
        p.category,
        DATE_FORMAT(o.order_date, '%Y-%m') AS month,
        SUM(oi.quantity * oi.unit_price) AS revenue
    FROM orders o
    JOIN order_items oi ON o.order_id = oi.order_id
    JOIN products p ON oi.product_id = p.product_id
    WHERE o.status = 'Delivered'
    GROUP BY p.category, month
),
with_growth AS (
    SELECT
        category,
        month,
        revenue,
        LAG(revenue) OVER (
            PARTITION BY category ORDER BY month
        ) AS prev_revenue
    FROM monthly_revenue
)
SELECT
    category,
    month,
    revenue,
    ROUND((revenue - prev_revenue) / prev_revenue * 100, 1) AS growth_pct
FROM with_growth
ORDER BY category, month;

🏥 Case Study 2: Hospital Patient Analytics

Business Problem: Hospital management wants to identify doctors with the highest patient load and average treatment time.

Solution Query
SELECT
    d.doctor_name,
    d.specialty,
    COUNT(DISTINCT p.patient_id) AS total_patients,
    AVG(DATEDIFF(a.discharge_date, a.admit_date)) AS avg_stay_days,
    SUM(CASE WHEN a.outcome = 'Recovered' THEN 1 ELSE 0 END) AS recoveries,
    ROUND(
        SUM(CASE WHEN a.outcome = 'Recovered' THEN 1 ELSE 0 END) /
        COUNT(*) * 100, 1
    ) AS recovery_rate
FROM doctors d
JOIN admissions a ON d.doctor_id = a.doctor_id
JOIN patients p ON a.patient_id = p.patient_id
GROUP BY d.doctor_id, d.doctor_name, d.specialty
ORDER BY total_patients DESC;

🏦 Case Study 3: Banking Fraud Detection

Business Problem: Identify suspicious transactions — accounts with multiple transactions in a short time window.

Solution Query
WITH txn_gaps AS (
    SELECT
        account_id,
        txn_id,
        amount,
        txn_time,
        LAG(txn_time) OVER (
            PARTITION BY account_id ORDER BY txn_time
        ) AS prev_txn_time
    FROM transactions
),
rapid_fire AS (
    SELECT *,
        TIMESTAMPDIFF(MINUTE, prev_txn_time, txn_time) AS mins_since_last
    FROM txn_gaps
    WHERE prev_txn_time IS NOT NULL
)
SELECT
    account_id,
    COUNT(*) AS rapid_transactions,
    SUM(amount) AS total_suspicious_amount
FROM rapid_fire
WHERE mins_since_last < 5   -- transactions within 5 minutes
GROUP BY account_id
HAVING COUNT(*) >= 3       -- 3+ rapid transactions
ORDER BY rapid_transactions DESC;
